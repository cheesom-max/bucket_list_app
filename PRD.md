# Product Requirements Document (PRD)

## Bucket List App

### 1. 개요

#### 1.1 제품 비전
사용자가 인생 목표를 체계적으로 관리하고 AI의 도움을 받아 실현할 수 있도록 돕는 웹 애플리케이션

#### 1.2 목표 사용자
- 인생 목표를 세우고 추적하고 싶은 개인
- 목표 달성을 위한 구체적인 계획이 필요한 사용자
- 여정을 기록하고 공유하고 싶은 사용자

#### 1.3 핵심 가치
- **체계적인 목표 관리**: 카테고리, 우선순위, 태그를 통한 효율적인 목표 분류
- **AI 기반 플래닝**: 목표 달성을 위한 단계별 계획 자동 생성
- **진행 상황 시각화**: 마일스톤과 진행률을 통한 동기 부여
- **여정 기록**: 미디어와 일기를 통한 경험 아카이빙

### 3. 비기능 요구사항

#### 3.1 성능
- 페이지 로드 시간: 3초 이내
- API 응답 시간: 500ms 이내 (AI 기능 제외)
- AI 응답 시간: 10초 이내

#### 3.2 보안
- HTTPS 필수
- 비밀번호 해시화 (bcrypt)
- JWT 기반 세션 관리
- CSRF 보호

#### 3.3 확장성
- 사용자당 버킷 아이템 무제한
- 동시 접속 사용자 1,000명 지원

#### 3.4 접근성
- 반응형 디자인 (모바일, 태블릿, 데스크톱)
- 키보드 네비게이션 지원
- 스크린 리더 호환

### 4. 데이터 모델

#### 4.1 핵심 엔티티

```
User
├── id, name, email, password
├── age, interests[]
└── bucketItems[]

BucketItem
├── id, title, description
├── category, priority, status
├── targetDate, completedAt
├── userId (FK)
├── plans[], progress[], media[], diary[], tags[]

BucketPlan
├── id, bucketItemId (FK)
├── steps (JSON), timeline (JSON)
├── resources (JSON), obstacles (JSON)

BucketProgress
├── id, bucketItemId (FK)
├── percentage, milestone, notes

BucketMedia
├── id, bucketItemId (FK)
├── url, type, caption

BucketDiary
├── id, bucketItemId (FK)
├── content, mood

Tag
├── id, name
└── bucketItems[] (M:N)
```

### 5. 사용자 플로우

#### 5.1 온보딩 플로우
1. 랜딩 페이지 접속
2. 회원가입 (이메일 또는 소셜)
3. 프로필 설정 (이름, 나이, 관심사)
4. AI 기반 버킷리스트 추천 받기
5. 대시보드로 이동

#### 5.2 버킷 아이템 생성 플로우
1. "새 버킷 추가" 클릭
2. 기본 정보 입력 (제목, 설명)
3. 카테고리, 우선순위, 목표일 설정
4. (선택) AI 계획 생성 요청
5. 저장 및 대시보드 반영

#### 5.3 진행 상황 업데이트 플로우
1. 버킷 아이템 상세 페이지 접속
2. "진행 상황 업데이트" 클릭
3. 진행률 입력, 마일스톤/노트 추가
4. (선택) 미디어 업로드
5. 저장 및 히스토리 반영
